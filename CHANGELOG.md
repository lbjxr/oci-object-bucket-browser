# Changelog

## v1.0.0 — 2026-04-22

这版已经可以当作一套可用的 OCI Object Storage 轻量 Web 面板来使用。

### 新增
- 单账号登录
- 对象列表与前缀过滤
- 下载
- 文本 / 图片 / PDF 预览
- 图片缩略图与文件图标
- 大文件 multipart 上传
- 上传进度、速度、ETA、取消、成功提示
- 上传会话恢复
- 单对象删除
- 批量删除

### 上传链路增强
- 默认分片调整为 `16 MB`
- 默认并发调整为 `6`
- 分片上传改为 `XMLHttpRequest`，进度更连续
- 速度显示改为最近 `3 秒` 滑动窗口平均
- 分片失败支持自动重试
- 重试策略按错误类型区分
- 上传接口返回更清楚的错误字段：`error_code` / `retryable` / `reason`
- 恢复上传时会对账 OCI 远端已上传 parts
- 远端对账失败时支持保守降级恢复，但最终合并前仍会再次严格校验

### 删除体验增强
- 支持多选批量删除
- 支持全选当前结果 / 清空选择 / 已选数量显示
- 删除成功对象会直接从当前列表移除
- 批量删除部分失败时会保留失败项并给出更清楚反馈
- 失败列表过长时支持折叠收纳

### 列表体验增强
- 对象列表补充了更清楚的元信息
- README 已整理为更短、更清爽的说明版本

### 修复
- 修复分片上传大 chunk 时进度体感卡顿的问题
- 修复上传会话并发写入导致的分片状态丢失问题
- 修复失败分片重试时污染已上传状态的问题
- 修复恢复上传时跳过已完成分片不稳定的问题
- 修复本地 session 与远端 multipart 状态漂移时的恢复问题
- 修复删除已上传测试文件时的前端可用性问题

### 实测结论
- `16 MiB chunk + 6 并发` 下，约 `382,938,704 bytes` 文件完成上传约 `130 秒`
- 平均吞吐约 `2.95 MB/s`
- 稳定阶段约 `3.5 MB/s`，峰值约 `4 MB/s`

### 当前边界
- 断点恢复仍是轻量方案，不是网盘级完整断点续传
- 重试策略已经更聪明，但还不是完整上传调度器
- 远端对账依赖 OCI 返回的 part 列表与 ETag
