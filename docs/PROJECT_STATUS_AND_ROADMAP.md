# Project Status and Roadmap

## 当前状态

`oci-object-bucket-browser` 现在已经是一套可用的轻量对象存储 Web 面板了。

适合：
- 自用 bucket 管理
- 小团队内部文件站
- OCI Object Storage 的轻量管理页

## 已完成

### 基础能力
- 单账号登录
- 对象列表
- 前缀过滤
- 下载
- 文本 / 图片 / PDF 预览
- 图片缩略图
- 文件类型图标
- 移动端友好布局
- systemd 部署

### 上传能力
- 小文件直传
- 大文件 multipart 上传
- 分片并发上传
- 上传进度、速度、ETA
- 上传取消
- 轻量恢复型断点续传
- 分片失败自动重试
- `429` 限流等待提示与退避重试
- 错误分类与更清楚的失败提示
- OCI 远端 parts 对账恢复
- 远端对账失败时的保守降级恢复

### 对象管理
- 单对象删除
- 批量删除
- 全选当前结果
- 清空选择
- 已选数量显示
- 删除成功对象局部移除
- 删除失败反馈增强
- 删除失败长列表折叠
- 更清楚的对象元信息展示

## 当前推荐默认值

- `APP_UPLOAD_CHUNK_SIZE_MB=16`
- `APP_UPLOAD_PARALLELISM=6`
- `APP_UPLOAD_SINGLE_PUT_THRESHOLD_MB=32`

## 当前边界

- 断点恢复还是轻量方案，不是完整网盘级续传
- 上传重试策略已经增强，但还可以继续细化
- 还没有批量下载
- 还没有重命名 / 移动
- 还没有目录树视图
- 还没有多用户权限体系

## 下一步建议

### P1
上传限流 / `429` 场景第二刀已补上：支持读取服务端等待时间并前端提示；multipart 上传在连续 `429` 时会保守地下调前端并发，并向用户展示当前已收敛到的并发数。后续若还要继续做，可再评估是否补更细的全局并发自适应与恢复策略。

### P2
删除失败项一键重试

### P3
批量下载

### P4
更强的目录感 / 目录树浏览

### P5
更丰富的对象元信息与预览能力

## 一句话总结

这项目现在已经过了“纯 MVP”阶段。

下一步重点不是补最基本功能，
而是继续做：
- 上传可靠性加固
- 删除体验打磨
- 对象管理增强
